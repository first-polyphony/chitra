# internal/

This folder holds maintainer-facing research, planning, and management documents — the kind of material that supports running the project but isn't meant to be part of the public documentation surface (`README.md`, `docs/`). It's not hidden, just clearly separated from what a public user of the repo needs to read. When this repo is eventually made public, review this folder's contents before the flip: some files may need to move elsewhere, be trimmed of any operator-personal details, or be deleted outright.
